

项目旨在打造一套支持 Android、iOS、Web、Mac、Windows 的跨平台云盘系统，客户端基于 KMP 构建，后端采用 Kotlin 实现。系统默认接入腾讯云 COS 作为对象存储底座，并通过统一的存储适配层实现对多云对象存储的可替换支持。产品一期优先交付 Android 与 Web，覆盖上传、下载、分享、文件管理、在线预览等核心能力，同时支持私有化部署，用户可通过配置自己的 API Key 与存储参数切换至自有 COS 或其他对象存储，实现灵活部署与后续扩展。

# 云盘项目 PRD

## 1. 项目概述

### 1.1 项目背景
随着企业数字化协作需求提升，文件的存储、分享、预览、协同管理逐渐成为组织内外协作的基础能力。为满足多端访问、私有化部署、云存储可替换、统一管理等需求，计划建设一套支持多平台的云盘系统。

本项目需满足以下核心约束：

1. 客户端采用 **KMP（Kotlin Multiplatform）** 跨平台技术。
2. 后端采用 **Kotlin** 开发。
3. 文件存储默认使用 **腾讯云 COS**，同时要支持后续切换其他对象存储服务。
4. 客户端支持 **Android、iOS、Web、Mac、Windows**，一期优先实现 **Android + Web**。
5. 核心功能包括：**上传、下载、分享、管理、预览** 等。
6. 支持 **私有化部署**：用户可通过更换 API Key / 存储配置切换到自己的 COS 或其他对象存储。

## 2. 产品目标

### 2.1 产品愿景
打造一套支持多端访问、易部署、可扩展、可替换存储后端的云盘系统，为个人或企业提供统一的文件资产管理能力。

### 2.2 产品目标
- 提供稳定可靠的文件存储与分发能力

- 为后续扩展权限、审计、协作、版本控制等能力预留架构基础

### 2.3 成功指标（建议）
- 上传成功率 ≥ 99.5%
- 下载成功率 ≥ 99.5%
- 文件列表首屏加载时间 ≤ 2s

- 分享链接可用率 ≥ 99.9%
- 多端核心能力一致性 ≥ 95%
- 私有化部署配置成功率 ≥ 95%

## 5. 产品定位与边界

### 5.1 产品定位
这是一个 **以对象存储为底座的多端文件管理平台**，不是单纯的网盘客户端，而是具备：
- 文件管理
- 文件分发
- 存储适配
- 私有化部署
- 跨端能力

### 5.2 设计原则
- **存储解耦**：业务层不得直接依赖具体云厂商 SDK
- **多端一致**：核心能力接口统一，UI 表现可按平台差异化
- **可配置化**：存储、域名、权限、过期策略等可配置
- **可扩展**：未来可增加版本管理、协作、搜索等模块
- **安全优先**：密钥、分享、鉴权、下载链接都要有安全边界

## 7. 核心模块设计

# 7.1 文件管理模块

### 功能描述
提供文件/文件夹的创建、查看、重命名、移动、删除、搜索、排序等管理能力。

### 功能点
- 文件列表展示
- 文件夹层级导航
- 新建文件夹
- 重命名
- 删除
- 移动
- 批量操作
- 按名称/时间/类型排序
- 按关键字搜索

### 规则说明
- 同一目录下文件名不可重复
- 删除支持软删除或回收站机制，MVP 可先做硬删除或逻辑删除
- 文件夹与文件操作需保持一致性
- 批量操作需提供失败回滚或部分失败提示

# 7.2 上传模块

### 功能描述
支持文件上传到对象存储，并写入云盘元数据。

### 功能点
- 单文件上传
- 多文件批量上传
- 大文件分片上传
- 断点续传
- 上传进度展示
- 上传失败重试
- 后台上传
- 上传前校验文件类型/大小

### 业务规则
- 单文件大小限制可配置
- 分片大小可配置
- 上传过程中需生成临时任务记录
- 上传成功后回写文件元数据与存储地址
- 支持秒传/去重能力（可作为二期优化）

### 上传流程建议
1. 客户端发起上传申请
2. 后端返回上传策略/临时凭证/分片信息
3. 客户端直传对象存储
4. 上传完成后通知后端完成入库
5. 后端生成文件记录

## 8. 私有化部署与多云适配设计

这是本项目的关键设计点，必须在 PRD 中明确。

## 8.1 私有化部署目标
支持用户在自己的环境中部署系统，并通过配置自己的对象存储账号实现完全独立使用。

### 支持的配置项
- 存储服务类型：COS / OSS / S3 / MinIO 等
- Region
- Bucket
- SecretId / SecretKey
- Access Key / Secret Key
- Endpoint
- 是否启用 CDN
- 预签名有效期
- 上传分片大小
- 回调地址

### 部署方式
- Docker Compose
- K8s
- 独立服务部署
- 后端配置文件方式部署

---

## 8.2 存储适配层设计

### 设计目标
避免业务层直接依赖腾讯云 COS SDK，后续切换对象存储厂商时仅替换适配器，无需修改核心业务逻辑。

### 建议抽象接口
统一定义对象存储接口，例如：

- `uploadObject`
- `uploadMultipart`
- `completeMultipart`
- `downloadObject`
- `deleteObject`
- `getObjectUrl`
- `createPresignedUrl`
- `copyObject`
- `listObjects`
- `getObjectMetadata`

### 分层建议
1. **Domain 层**
   - 定义对象存储能力接口
2. **Infrastructure 层**
   - 实现 COS Adapter
   - 后续实现 OSS/S3 Adapter
3. **Service 层**
   - 编排上传、下载、元数据写入
4. **API 层**
   - 对外暴露统一接口

### 关键原则
- 业务只认“对象存储能力”，不认具体厂商
- 厂商差异统一封装在适配器中
- 所有 SDK 初始化参数通过配置注入
- 上传/下载使用统一回调与错误码模型

---

## 8.3 私有化部署安全要求
- 密钥不明文存储
- 支持环境变量注入
- 配置文件支持加密字段
- 日志中禁止打印密钥
- 支持配置轮换
- 支持多租户隔离
- 支持权限最小化原则

## 11. 关键业务流程

## 11.1 上传流程
1. 用户选择文件
2. 客户端向后端申请上传任务
3. 后端返回签名或临时上传信息
4. 客户端直传对象存储
5. 上传完成后通知后端
6. 后端落库文件元数据
7. 前端刷新文件列表

## 11.2 下载流程
1. 用户点击下载
2. 后端校验权限
3. 后端返回签名下载地址或代理下载
4. 客户端开始下载并展示进度
5. 下载完成记录日志

## 11.3 分享流程
1. 用户选择文件生成分享
2. 配置过期时间/密码/权限
3. 后端生成分享 token
4. 外部用户通过链接访问
5. 后端验证规则后返回预览/下载能力

## 11.4 私有化切换流程
1. 管理员进入存储配置页
2. 填写新的 COS/OSS/S3 配置
3. 点击测试连接
4. 连接成功后启用该存储
5. 新文件走新存储策略
6. 可选择保留历史存储只读访问

## 15. 与腾讯云 COS 的关系说明

当前项目默认采用腾讯云 COS 作为对象存储基础设施，适合作为一期快速落地方案。

你提供的参考文档包括：
- 腾讯云对象存储服务官网
- Android SDK
- iOS SDK
- Java SDK

但在产品设计上，不应把 COS 视为“业务强绑定”，而应将其作为 **默认实现**。  
产品和架构上要明确：

- COS 是当前默认供应商
- 存储能力通过适配层封装
- 后续可切换为 S3 兼容存储或其他云厂商
- 私有化部署时可切换到用户自己的 COS 账号与桶

## 16. 风险与对策

### 风险 1：对象存储厂商强绑定
**问题**：业务代码直接依赖 COS SDK，后续切换成本高。  
**对策**：必须建立统一的存储接口与适配器层。

### 风险 2：多端 UI 差异大
**问题**：KMP 能共享逻辑，但各平台交互差异较大。  
**对策**：共享领域层与数据层，UI 按平台适配。

### 风险 3：大文件上传失败
**问题**：网络不稳定导致上传中断。  
**对策**：分片、断点续传、重试、后台任务。

### 风险 4：分享链接安全性不足
**问题**：链接泄露导致文件外泄。  
**对策**：提取码、有效期、访问控制、审计日志。

### 风险 5：私有化部署配置复杂
**问题**：用户切换存储配置失败。  
**对策**：提供连接测试、配置向导、默认模板、错误提示。
