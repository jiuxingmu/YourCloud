# YourCloud Web UI 重设计方案

## 1. 背景与目标

当前 Web UI 功能可用，但整体仍偏“后台表格页”体验：

- 文件浏览依赖高密度表格
- 预览区域不是视觉中心
- 分享/下载反馈与操作位置分散
- 登录页较通用，品牌识别弱

本次改版将同步调整视觉风格、页面布局和核心交互。

核心目标：

- 在保留文件管理效率的前提下，升级为更具“作品集感”的体验。

## 2. UX 目标

落地 “Editorial Split（编辑式双栏）” 体验：

- 创意、展陈感的视觉语言
- 左侧文件目录 + 右侧预览舞台作为默认信息架构
- 预览优先的交互模型
- 一致的状态反馈和响应式行为

## 3. 范围

### 本次包含

- `clients/web/src/App.tsx`：全局外壳与页面节奏
- `clients/web/src/pages/LoginPage.tsx`：登录页分栏布局与交互更新
- `clients/web/src/pages/FilesPage.tsx`：布局、交互与状态的整体重构
- 在现有 MUI 结构内统一视觉 token 和风格

### 本次不包含

- 后端 API 协议调整
- 新增后端预览/转码能力
- 超出前端启发式范围的完整文件类型语义识别
- 当前应用中不存在的用户资料/设置页

## 4. 方向选择与备选方案

### 已选方案：方案 1 - Editorial Split

选择原因：

- 在视觉气质与操作清晰度之间平衡最好
- 天然支持“预览优先”路径
- 对文件管理用户仍保留熟悉感

核心思路：

- 左栏为“策展式文件目录”
- 右栏为“持续可用的预览舞台”
- 关键操作锚定在预览上下文中

### 备选方案：Gallery-first Hybrid

- 视觉冲击更强
- 对文档型文件流程支持较弱

### 备选方案：Minimal Studio

- 极简高级感更明显
- 新用户操作可发现性更低

## 5. 信息架构与布局

## 5.1 全局外壳

- 顶部保留 App Bar，但简化为品牌 + 账户控制
- 使用全高内容框架（`100vh` 扣除顶部栏）
- 在桌面端减少“中间漂浮卡片感”，采用更贴边但有节制的宽度控制

## 5.2 登录页

- 桌面端改为左右双栏：
  - 左侧：品牌宣言、价值点、装饰性背景
  - 右侧：认证卡 + Tab 切换（`登录` / `创建账号`）
- 移动端改为单列堆叠（品牌区在上，认证卡在下）
- 统一一个反馈区域承载加载、成功、错误信息

## 5.3 文件页

桌面端（`>=1024px`）：

- 12 栅格双栏：
  - 左侧目录区：4 列
  - 右侧预览区：8 列

平板端（`768-1023px`）：

- 5/7 分栏，保持预览主导

移动端（`<768px`）：

- 上下布局：
  - 先文件目录
  - 后预览区域
  - 操作区与预览紧密编组

## 6. 交互设计

## 6.1 选择与预览（P0）

- 文件加载成功后：
  - 若有文件则自动选中第一个
- 左侧目录支持：
  - 点击选择
  - 键盘上下切换
  - `Enter` 将焦点移动到右侧预览操作区
- 右侧预览优先级：
  - 图片：图片预览
  - 视频：内嵌播放器
  - 其他：强化信息占位卡（文件信息优先）
- 切换文件时仅更新右侧预览区域，并做轻量淡入过渡

## 6.2 核心操作（P0）

- 右侧预览区操作层级：
  1. 下载（主按钮）
  2. 创建分享链接（次按钮）
  3. 复制链接（链接生成后展示）
- 分享链接展示移入预览上下文区，不再放在全局顶部流程
- 成功/失败反馈统一为同一套瞬态提示模式

## 6.3 上传流程（P1）

- 上传入口保留在左侧目录头部
- 同时支持点击上传与拖拽上传
- 上传完成后：
  - 刷新文件列表
  - 在可确定时自动定位并选中新上传文件

## 6.4 状态处理（P0）

- 空状态：
  - 左侧：无文件引导
  - 右侧：插画感占位 + 单一明确 CTA
- 加载状态：
  - 左侧 skeleton 列表
  - 右侧 skeleton 预览舞台
- 错误状态：
  - 尽量就近展示到触发操作附近
  - 对于非局部错误保留全局兜底提示

## 7. 视觉语言

- 基调：创意作品集 / 编辑工作室感，而非企业后台感
- 色彩：
  - 柔和中性背景 + 高对比强调色（蓝紫系）
- 容器：
  - 更柔和圆角、轻边框、克制阴影
- 字体层级：
  - 标题更有节奏，辅助信息更干净
- 动效：
  - 短时过渡（120-180ms），避免复杂编排

## 8. 可访问性与可用性要求

- 键盘可用性：
  - 所有可交互元素有清晰 focus-visible
  - 文件目录可脱离鼠标完成操作
- 可读性：
  - 主文本和关键操作保持足够对比度
- 反馈可感知：
  - 加载与结果反馈均可被用户明确感知
- 响应式一致性：
  - 移动端仍可顺畅完成“选择 -> 预览 -> 下载/分享”主路径

## 9. 验收标准

- 用户在 3 秒内可识别主流程：
  - 选文件、看预览、执行操作
- 桌面端预览区始终是视觉中心
- 操作反馈使用统一模式与一致位置逻辑
- 登录页具备明确品牌表达与更清晰的认证决策流程
- 移动端可完整完成上传、预览、下载、分享且无隐藏关键控件

## 10. 实施备注（供下一步计划使用）

- 建议从 `FilesPage` 提取小型 UI 子组件，降低复杂度：
  - 目录区组件
  - 预览舞台组件
  - 操作栏组件
  - 状态反馈组件
- 保持 API 使用方式不变，将改动集中在表现层和局部交互状态
- 实现后通过本地 `npm` 测试/构建流程做回归验证
