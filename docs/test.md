# 测试经验与改进清单

更新时间：2026-04-28

## 这次暴露的核心问题

- 单元测试通过，但缺少启动级与集成级验证，导致 `JWT_SECRET` 配置变更后本地无法登录。
- 测试偏重 happy path，边界场景覆盖不足（未来时间、目录移动到子目录、可伪造请求头）。
- 前后端契约与错误语义未被系统化校验，出现“能跑但语义错/风险高”的问题。

## 后续改代码的硬规则

1. **无启动验证，不允许宣称修复完成。**
2. **涉及鉴权/配置/路由变更，必须补对应回归测试。**
3. **涉及前后端交互，必须验证契约（方法、参数、响应结构、错误码）。**
4. **涉及目录树与时间筛选，必须补边界测试。**
5. **安全问题优先级高于功能优化，先修 P0/P1 再做体验改动。**

## 必测清单（每次发布前）

### A. 启动与配置

- [ ] `./scripts/dev.sh` 可成功启动 API + Web + DB。
- [ ] `GET /health` 返回 200。
- [ ] `JWT_SECRET` 为空时启动失败（符合安全预期）。
- [ ] 本地开发存在兜底机制（如 `dev.sh` 注入）保证可运行。

### B. 鉴权与登录链路

- [ ] 注册、登录、`auth/me` 全链路通过。
- [ ] 无 token / 伪造 token 必须返回 401。
- [ ] 更换 `JWT_SECRET` 后旧 token 失效行为符合预期（需重新登录）。

### C. 文件核心能力

- [ ] 上传、下载、分享、移动文件通过。
- [ ] 文件夹新建、删除、移动通过。
- [ ] 禁止“目录移动到自身或自身子目录”。
- [ ] 目录删除后子项与存储数据一致清理。

### D. 分享与安全

- [ ] 分享提取码校验通过（读取/下载/缩略图）。
- [ ] 分享 URL 不信任客户端 `Origin`。
- [ ] 过期/不存在/提取码错误的状态码与前端提示一致。

### E. 前端筛选与展示

- [ ] “最近”分组不误判未来时间。
- [ ] 7d/30d 筛选不包含未来时间数据。
- [ ] 推荐列表与产品规则一致（如只展示文件，不展示文件夹）。

## 测试分层要求

- **单元测试**：覆盖函数级逻辑与边界。
- **集成测试**：覆盖 handler/service/repo 组合行为。
- **启动冒烟测试**：覆盖真实启动与关键接口可用性。
- **端到端冒烟**：覆盖登录后核心用户路径。

> 最低要求：每次发布前至少执行一轮“启动冒烟 + 核心链路 E2E 冒烟”。

## 变更类型与必补测试映射

- 改 `config` / `middleware`：补配置校验与鉴权回归。
- 改 `handler`：补接口状态码/错误码测试。
- 改 `service`：补业务规则与边界测试。
- 改前端 `domain/selectors`：补纯函数边界测试。
- 改前端 API 调用：补方法/路径契约测试或联调验证。

## 发布门禁（Go / No-Go）

### Go

- P0 缺陷为 0。
- P1 缺陷有明确风险接受记录与排期。
- 必测清单全部通过并有日志证据。

### No-Go

- 任一 P0 未关闭。
- 启动冒烟未通过。
- 鉴权链路异常（登录、token 校验、权限控制）。
